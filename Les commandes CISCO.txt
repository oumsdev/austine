Permert de gerer les VLANs
Pre-requis : trunker les liaisons entre les switchs 
Passer le switch cœur en mode vtp server et les autres en mode vtp client 
Sur le Switch Server 
Enable 
Conf t 
Vtp mode server 
Vtp domain tssr 
Vtp password cisco 
Sur les Switch client 
Enable 
Conf t 
Vtp mode client 
Vtp domain tssr
Vtp password cisco
Ensuite on va sur le switch server, on déclare ses vlan

NAT STATIQUE 
Associer une adresse Privée à 1 adresse Publique.  
1 seul machine peut utiliser cette adresse 
Ici on veut associer au serveur 192.168.1.150 l’adresse publique 201.49.10.17 
Enable 
Conf t

NAT DYNAMIQUE 
Plusieurs machines privées vont se partager la ou les mêmes adresses publiques 
Attention les machines ne pourront pas sortir en même temps car il faut que les adresses publiques 
ne soit pas utilisées

Conf t 
!!Etape D.1 => On créer son étendu

192.168.1.150)

!!Etape D.3 => On applique le NAT Dynamique 
Ip nat inside source list 22 pool POOL-PURPLE-NAT
